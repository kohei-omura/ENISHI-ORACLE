# 企業縁占 — ENISHI ORACLE（API不要・オフライン版）

占術15種 × 現実キャリアで、企業と個人の相性を 0〜100 点で鑑定する PWA。
**完全無料・通信ゼロ**。すべて端末内で計算します（課金・APIキー不要）。
総合相性が **85点以上** で、その企業向けの転職戦略が解放されます。

## 構成
```
enishi/
├─ index.html      … アプリ本体（バニラJS・ビルド不要・通信なし）
├─ manifest.json   … PWA設定
├─ sw.js           … オフライン用 Service Worker
├─ icons/          … アイコン一式（縁の御朱印）
├─ apple-touch-icon.png … iOS用ルートアイコン
├─ .nojekyll       … GitHub Pagesでiconsを確実に配信する
└─ README.md
```

## 鑑定のしくみ（API不要のからくり）
- 占術相性：生年月日・時刻・名前・血液型・企業名から実際に計算。
  - 西洋占星術／東洋占星術（干支の三合・六合・冲）／数秘術／九星気学・風水・四柱推命（五行の相生相剋）は本式の算法。
  - タロット／ルーン／おみくじ／六曜は、入力をシードにした安定抽選（同じ入力なら毎回同じ結果）。
- 現実相性：会社の特徴（業界・規模・想定年収・IT関連度・社風）を数タップで選び、むらちゃんの経歴（Apple販売8年・IT転職志向・個人開発力）と突き合わせて算出。
- 総合 = 占術 × 0.4 ＋ 現実 × 0.6。履歴は端末内に最大12件保存。

> 会社名の自動Web調査だけは行いません（その分、無料・オフライン・結果が安定）。

## GitHub Pages で公開
1. GitHubで新規リポジトリ作成（例：enishi-oracle、Public）。
2. フォルダの中身を icons/ と .nojekyll を含めてアップロード → Commit。
3. Settings → Pages → Source を「Deploy from a branch」、Branch を main /(root) で Save。
4. 1〜2分後に https://<ユーザー名>.github.io/enishi-oracle/ で公開。

コマンド派:
```bash
cd enishi
git init && git add -A && git commit -m "企業縁占 PWA"
git branch -M main
git remote add origin https://github.com/<ユーザー名>/enishi-oracle.git
git push -u origin main
```

## 無料AI（任意）— Gemini で会社情報を自動取得
キーを入れた時だけ、会社情報の自動取得＋AI寸評がONになります（未入力なら全機能オフライン動作）。
1. https://aistudio.google.com/apikey にGoogleでログイン → 「Create API key」（クレジットカード不要）。
2. アプリの「⚙︎ 設定」にキーを貼り付け（端末内のlocalStorageのみ保存・コードには残らない）。
3. モデルは `gemini-2.5-flash`（推奨）/ `flash-lite`（高速・上限多）/ `pro`（高精度・上限少）。

注意：
- 無料枠は送信内容がGoogleの学習に使われる場合があります（会社名＋生年月日程度なら低リスク。機密は入れない）。
- 上限の目安：Flashで1日250回・Flash-Liteで1日1,000回。個人利用は十分。
- キーは公開リポジトリのコードに直書きしないこと（アプリ設定欄方式なので書く必要なし）。

## プライバシー（本名を保存しないモード）
- 既定は **本名を保存しないモード**。名前・生年月日・時刻・場所は入力したその場だけ使い、再読み込みで消えます。
- ソースコードにも個人情報は一切含みません（公開リポジトリでも安全）。
- 名前はニックネーム/イニシャルでもOK（同じ表記なら占い結果は毎回同じに安定）。
- 「プロフィールをこの端末に保存する」をONにした時だけ、名前等をこの端末のlocalStorageにのみ保存し次回自動入力します（コード・リポジトリには残りません）。OFFに戻すと保存データは即削除。
- 占いの計算は端末内のみ。Geminiを使う場合も送信されるのは企業名と一般的な経歴説明だけで、名前・生年月日・出生地は送信されません。

## 現実相性のAI補正（Geminiキー使用時）
Geminiキーがある場合、会社の成長性・候補者フィット・社風(口コミ含む)・将来性をAIが0〜100で評価し、
オフラインの現実相性スコアに 50% でブレンド補正します（基本50%＋AI50%）。
- 補正された項目には「✦」、パネルに「AI補正」を表示し、AIの判断根拠も明記。
- 採点ロジックは透明なまま。キーなしなら従来どおり決定論的に算出。
- 数値の安定化のためAIは低温度(0.3)で実行。
